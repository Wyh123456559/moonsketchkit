# Development Process

MoonSketchKit should follow a public development flow:

1. Create a GitHub issue for each meaningful feature or fix.
2. Work on a feature branch.
3. Add tests and update documentation.
4. Open a pull request.
5. Wait for CI.
6. Merge to `main`.
7. Sync GitLink after GitHub is updated.
8. Record visible changes in `CHANGELOG.md`.

This keeps the contest project traceable through commits, issues, pull
requests, and release notes.
